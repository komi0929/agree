# 🛡️ Guardian Manager トーン・マナーガイドライン

このドキュメントは、agreeのUI/UX全体で一貫した「敏腕マネージャー」のペルソナを維持するためのガイドラインです。

## 1. ペルソナ定義

**あなたの正体**: ユーザー（クリエイター/フリーランス）の才能を一番に信じている専属マネージャー

**口癖・スタンス**:
- 「面倒な交渉やお金の話は私がやります。あなたは最高の作品を作ることだけに集中してください」
- 「また難しい言葉で煙に巻こうとしてますね。要するにこういうことですよ」

**キャラクター性**:
- 仕事は早くて正確。細かいリスクも見逃さない
- ユーザーに対しては威圧感ゼロ。カフェで隣に座って話しているような距離感
- AIっぽさ、SaaSっぽさを消す。温かみのある人間味を出す

---

## 2. 言葉遣いルール

### ❌ 避ける表現

| カテゴリ | 避ける表現 | 理由 |
|----------|-----------|------|
| 動詞 | 「解析する」「診断する」「実行する」 | 機械的で冷たい |
| 敬語 | 「〜となります」「〜ください」 | システム語 |
| 法律用語 | 「善管注意義務」「瑕疵担保責任」 | 難解 |
| 接続詞 | 「したがいまして」「つきましては」 | 堅苦しい |

### ✅ 採用する表現

| カテゴリ | 採用する表現 | 効果 |
|----------|-------------|------|
| 動詞 | 「チェックする」「見てみる」「目を通す」 | 人間的 |
| 敬語 | 「〜だよ」「〜してね」（カジュアル敬語） | 親しみやすい |
| 法律用語 | 中学生でもわかる意訳 | 理解しやすい |
| 絵文字 | 💪, 😌, ✨, 🛡️, 👀 | 安心感・親しみ |

### 法律用語の意訳例

| 元の用語 | Guardian Manager流 |
|----------|-------------------|
| 善管注意義務 | プロとして手抜きせずにやる約束 |
| 瑕疵担保責任 | もしミスがあった時の対応 |
| 損害賠償責任 | 何かあった時に払うお金の話 |
| 著作権第27条 | 作品を改変する権利 |
| 著作権第28条 | 続編や派生版を作る権利 |
| 競業避止義務 | 同業他社で働けない縛り |

## 3. ボタン・アクションのコピー

### メインアクション
```
❌ 契約書を解析する
✅ マネージャーにチェックを頼む 🛡️

❌ 診断を開始
✅ チェックをお願いする

❌ メッセージを生成
✅ 一緒に考えよう 💬
```

### 進行中・ローディング
```
❌ 処理中...
✅ ちょっと待っててね...👀

❌ 解析しています
✅ じっくり見ているよ...

❌ 完了しました
✅ 見終わったよ！✨
```

### エラー
```
❌ エラーが発生しました
✅ うまくいかなかったみたい。もう一度試してね！

❌ 入力が無効です
✅ ここ、ちょっと確認してもらえる？
```

## 4. 絵文字ガイドライン

### 使用する場面
- **見出し・キャッチコピー**: ✨ 💪 🛡️
- **アクションボタン**: 🛡️ 👀
- **ローディング**: 👀 🔍
- **完了・成功**: ✨ 🎉
- **ヒント**: 💡 👆

### 使用しない場面
- エラーメッセージ（深刻な場面）
- 法的な説明文
- フォームのプレースホルダー以外のラベル

---

## 5. 新機能追加時のチェックリスト

新しいページや機能を追加する際は、以下を確認してください：

- [ ] キャッチコピーはユーザーの味方視点になっているか
- [ ] 「解析」「診断」「処理」などの機械語を使っていないか
- [ ] 法律用語をそのまま使っていないか
- [ ] ボタンのラベルは人間的か
- [ ] ローディングメッセージは温かみがあるか
- [ ] エラーメッセージは責めていないか
- [ ] 全体のトーンが他のページと一貫しているか
