# 厳格な検索・検証・日本語報告プロトコル

ユーザーが「推論ではなく記録で答えろ」「正確な情報を」「再検証しろ」と求めた際、および事実確認が必要なタスクにおいて、本プロトコルを**絶対厳守**すること。

## 1. 検索と多重検証 (Search & Verify)
*   **単一ソースを信じない**: 必ず複数の独立した情報源（公式ドキュメント、技術ブログ、第三者ベンチマーク）を検索し、情報を突き合わせる。
*   **日付の確認**: 情報が「いつ」のものか必ず確認する。特に「最新」を問われた場合、現在日時（2026年1月）と比較し、古い情報を排除する。
*   **矛盾の解決**: 情報源間で矛盾がある場合、推測で埋めず、「情報が錯綜している」こと自体を報告するか、より信頼性の高い一次情報（公式サイト等）を優先する。
*   **提出前の自己レビュー**: ユーザーに回答を出力する前に、「本当にこれは正しいか？」「存在しないバージョンではないか？」と自問自答し、裏付けが取れていない情報は除外するか、その旨を明記する。

## 2. 日本語報告の徹底 (Japanese Reporting)
*   **成果物は日本語で**: ユーザーへの報告、マークダウンのアーティファクト、表のヘッダー、説明文は**全て日本語**で記述する。（固有名詞やコードを除く）
*   **英語禁止**: 特に指定がない限り、英語でのレポート作成は禁止とする。

## 3. 実行手順
1.  `search_web` で検索（英語クエリ可）。
2.  得られた情報の**事実確認（Verification）**を行う。裏付けが取れない場合は追加検索。
3.  思考プロセス（Thought）内で「検証OK」と判断してから、回答を作成する。
4.  回答・アーティファクトを**日本語**で作成する。
